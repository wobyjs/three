# Phase 19 Plan 02: Animation / Skinning Loader Batch

## Context

Middle plan of Phase 19. 19-01 covers 3D-model-format loaders (3DTiles, IFC, LDraw, USDZ). This plan groups 4 animation-and-skinning loader demos that all exercise the `AnimationMixer` + per-frame `mixer.update(delta)` pattern with the loader filling in skinned/morphed/kinematic mesh data:

- **Collada kinematics**: ColladaLoader's kinematics rig (industrial-robot articulated arm)
- **Collada skinning**: ColladaLoader's skinned-mesh + `AnimationMixer` (avatar/character)
- **FBX NURBS**: FBXLoader's NURBS-curve → tessellated BufferGeometry conversion
- **MD2 control**: `MD2CharacterComplex` from `examples/jsm/misc/MD2CharacterComplex.js` — Quake-2 morphTarget character driven by WASD keys

All 4 IDs were **verified absent from `demo/src/registry.ts`** at planning time (2026-06-26 grep, see 19-CONTEXT.md "Pre-Flight Scope Verification"). All 4 IDs **confirmed present in upstream `threejs.org/examples/files.json`**. All 4 have **pre-converted `.tsx` scaffolds** under `code/examples/webgl/loaders/` (table in 19-CONTEXT.md) — scaffolds are stub-quality per Phase-15/17/18 precedent; implementer MUST consult upstream HTML.

## Locked Decisions (from Phase 15 RESEARCH / Phase 17 + 18 SUMMARYs)

- **init3D pattern**: `const init3D = (container: HTMLElement) => { ... }` with module-level `let _cleanupFn`. JSX wraps with `<div ref={(el) => { if (el) init3D(el) }} />`.
- **JSX pragma**: every file begins with `/** @jsxImportSource woby */`.
- **Forbidden**: `as any`, `useEffect`, `useRef` for plain objects.
- **Visual verification**: dv profile-4 screenshot → Claude approval → THEN register.
- **Dev server**: http://localhost:5300, profile-4 only.
- **Asset paths**: `https://threejs.org/examples/models/...`. Substitutions allowed when documented.
- **AnimationMixer cleanup**: `mixer.stopAllAction()` + `mixer.uncacheRoot(root)` in `_cleanupFn` to prevent memory leak.
- **Keyboard listeners (Task 4)**: bind on `window`, capture references, and `removeEventListener` in cleanup.

## Risk Notes

- **Collada kinematics**: ABB robot model is small (~1MB) and reliable. TWEEN-based joint randomization may render as static if the screenshot is taken at the exact rest moment — re-take if needed.
- **Collada skinning**: The Stormtrooper model is the canonical safe choice; verify the upstream `.dae` URL is still live. If not, substitute another skinned Collada from `threejs.org/examples/models/collada/`.
- **FBX NURBS**: NURBS curves render as thin polylines that can be hard to see against a white background. Match upstream's background color (typically `0xa0a0a0` or similar mid-gray).
- **MD2 control**: MD2CharacterComplex shares JSON-config-driven multi-skin loading; the bundled JSON may reference textures with case-sensitive paths that fail on strict servers. Watch console for 404s and fall back to a single-skin model if needed.
- **Animation mixer leaks**: forgetting `mixer.stopAllAction()` + `mixer.uncacheRoot(root)` in cleanup leaks the entire scene graph between demo navigations — strictly enforce.
