# Phase 16 Plan 01: Marching Cubes Rewrite + Textures Batch 1

## Context

Phase 15 ended at 589 registered demos. Current registry has 675 IDs (additional work done outside GSD phases). This plan ports 6 more demos using the established init3D pattern.

## Locked Decisions (from Phase 15 RESEARCH)

- **init3D pattern**: `const init3D = (container: HTMLElement) => { ... }` with module-level `let _cleanupFn`
- **JSX pragma**: `/** @jsxImportSource woby */` (NOT `@woby/three`)
- **Visual verification**: dv CLI screenshot → Claude approval → then register
- **No `as any`**, no `useEffect`, no `useRef` plain objects
- **Dev server**: http://localhost:5300, profile-4

## Tasks

### Task 1: Rewrite WebGLMarchingCubes

**File**: `demo/src/WebGLMarchingCubes.tsx`
**Issue**: Currently uses `Canvas3D` from `@woby/three` + imperative setup at module level — broken pattern
**Fix**: Rewrite entirely to init3D pattern
**Key features**:
- MarchingCubes from `three/examples/jsm/objects/MarchingCubes.js`
- Animated metaballs (4 blobs in sine-wave pattern)
- PhongMaterial with shininess
- OrbitControls
- GUI with resolution, isolation, opacity, color
**Registry ID**: `webgl_marching_cubes`
**Category**: `advanced`

### Task 2: Port WebGLTexturesEnvmap

**File**: `demo/src/WebGLTexturesEnvmap.tsx`
**Source**: threejs.org/examples/#webgl_textures_envmap
**Key features**:
- `CubeCamera` for real-time environment capture
- Moving sphere reflects its environment
- PMREMGenerator for environment lighting
- OrbitControls
**Registry ID**: `webgl_textures_envmap`
**Category**: `textures`

### Task 3: Port WebGLTexturesRotate

**File**: `demo/src/WebGLTexturesRotate.tsx`
**Source**: threejs.org/examples/#webgl_textures_rotate
**Key features**:
- Texture loaded on a plane
- `texture.rotation` animated in render loop
- `texture.center.set(0.5, 0.5)` for center rotation
- GUI control for rotation speed
**Registry ID**: `webgl_textures_rotate`
**Category**: `textures`

### Task 4: Port WebGLTexturesChecker

**File**: `demo/src/WebGLTexturesChecker.tsx`
**Source**: threejs.org/examples/#webgl_textures_checker
**Key features**:
- Canvas-generated 2-color checker pattern
- `CanvasTexture` from canvas 2D API
- Applied to boxes with different anisotropy settings
- Shows texture filtering comparison
**Registry ID**: `webgl_textures_checker`
**Category**: `textures`

### Task 5: Port WebGLTexturesDisplacement

**File**: `demo/src/WebGLTexturesDisplacement.tsx`
**Source**: threejs.org/examples/#webgl_textures_displacement (or webgl_materials_displacementmap)
**Key features**:
- Displacement map on a subdivided sphere
- `MeshStandardMaterial` with `displacementMap`, `displacementScale`
- Texture loaded from three.js CDN or procedural
- OrbitControls
**Registry ID**: `webgl_textures_displacement`
**Category**: `textures`

### Task 6: Port WebGLTexturesGradients

**File**: `demo/src/WebGLTexturesGradients.tsx`
**Source**: threejs.org/examples/#webgl_textures_gradients
**Key features**:
- Canvas-generated gradient (linear gradient via ctx.createLinearGradient)
- `CanvasTexture` wrapping canvas
- Applied to a torus or sphere
- Minimal controls or none
**Registry ID**: `webgl_textures_gradients`
**Category**: `textures`
