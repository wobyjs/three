# Phase 16 Plan 03: Shaders + Texture-Loader Batch

## Context

Final plan of Phase 16. Originally scoped to include 5 demos with `webgl_shaders_ocean` or `webgl_shaders_sky`, but both are already registered (`registry.ts` lines 598–599 and 611–613). A tonemapping-shader substitute was also rejected as a duplicate of registered `webgl_tonemapping` (line 600). **Resolution**: drop the shader slot and keep this as a focused 4-demo plan — 1 screenspace shader pass + 3 texture-loader format demos (TGA, KTX2, WebP) that round out the loader-texture family already covered by `webgl_loader_texture_exr/hdr/dds/basis`. All 4 IDs verified absent from `demo/src/registry.ts` at planning time.

- **init3D pattern**: `const init3D = (container: HTMLElement) => { ... }` with module-level `let _cleanupFn`

## Tasks

### Task 1: Port WebGLPostprocessingScreenspace

**File**: `demo/src/WebGLPostprocessingScreenspace.tsx`
**Source**: threejs.org/examples/#webgl_postprocessing_screenspace (or analogous screen-space distortion shader)
**Key features**:
- `EffectComposer` from `three/examples/jsm/postprocessing/EffectComposer.js`
- `RenderPass` for the base scene (a `TorusKnotGeometry` with `MeshNormalMaterial` rotating)
- Custom `ShaderPass` using a `ShaderMaterial`-style shader object with `uniforms: { tDiffuse, time, resolution }`, `vertexShader` (standard fullscreen-quad UV passthrough), `fragmentShader` doing screen-space ripple/distortion: `vec2 uv = vUv + 0.02 * vec2(sin(uv.y*40.0 + time), cos(uv.x*40.0 + time));`
- `OutputPass` last
- Animation loop updates `time` uniform; resize updates `resolution` uniform and `composer.setSize`
- OrbitControls on the base camera
**Registry ID**: `webgl_postprocessing_screenspace`
**Category**: `postprocessing`

### Task 2: Port WebGLLoaderTextureTGA

**File**: `demo/src/WebGLLoaderTextureTGA.tsx`
**Source**: threejs.org/examples/#webgl_loader_texture_tga
**Key features**:
- Import `TGALoader` from `three/examples/jsm/loaders/TGALoader.js`
- Two `PlaneGeometry` quads side by side, each with a `MeshBasicMaterial`
- Left quad: TGA texture loaded with `new TGALoader().load('texture.tga')` (use a known-public TGA URL such as `https://threejs.org/examples/textures/crate_grey8.tga` or commit a small fixture)
- Right quad: same image as PNG/JPG for comparison (optional — at minimum show the TGA renders correctly)
- Both textures use `texture.colorSpace = SRGBColorSpace`
- Slow OrbitControls or a fixed front-facing camera
**Registry ID**: `webgl_loader_texture_tga`
**Category**: `textures`

### Task 3: Port WebGLLoaderTextureKTX2

**File**: `demo/src/WebGLLoaderTextureKTX2.tsx`
**Source**: threejs.org/examples/#webgl_loader_texture_ktx2
**Key features**:
- Import `KTX2Loader` from `three/examples/jsm/loaders/KTX2Loader.js`
- `const ktx2Loader = new KTX2Loader().setTranscoderPath('https://unpkg.com/three@<version>/examples/jsm/libs/basis/').detectSupport(renderer)` — pass the live `renderer` so format detection uses real GPU caps
- Load a sample `.ktx2` (Basis-compressed) texture from three.js examples textures dir
- Apply to a `PlaneGeometry` or rotating `BoxGeometry`
- Show a fallback console.warn if KTX2 not supported (older WebGL1-only context)
- `ktx2Loader.dispose()` on cleanup
**Registry ID**: `webgl_loader_texture_ktx2`
**Category**: `textures`

### Task 4: Port WebGLLoaderTextureWebP

**File**: `demo/src/WebGLLoaderTextureWebP.tsx`
**Source**: threejs.org/examples/#webgl_loader_texture_webp
**Key features**:
- Standard `TextureLoader().load('image.webp', ...)` — no special loader needed; browsers decode WebP natively
- Apply to a rotating `BoxGeometry` or `PlaneGeometry`
- `texture.colorSpace = SRGBColorSpace`
- Optional: load both a `.webp` and a `.jpg` of the same image side-by-side to show no visible quality loss
- Animation: slow rotation; minimal GUI
- Picks a public WebP URL (e.g., `https://threejs.org/examples/textures/memorial.webp` or similar) — if not available, commit a small WebP fixture under `demo/public/textures/`
**Registry ID**: `webgl_loader_texture_webp`
**Category**: `textures`
