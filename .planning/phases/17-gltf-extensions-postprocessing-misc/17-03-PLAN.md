# Phase 17 Plan 03: Postprocessing Batch

## Context

Final plan of Phase 17. 17-01 covers 4 GLTF material extensions; 17-02 covers 4 advanced GLTF loaders. This plan rounds out the phase with 4 postprocessing demos that exercise distinct EffectComposer pipelines: env-background compositing, channel-split halftone, fullscreen procedural rendering, and layer-selective bloom. Grouped together because they all use the same `EffectComposer` + `RenderPass` + `ShaderPass` foundation, varying only in the specific pass logic.

All 4 IDs verified absent from `demo/src/registry.ts` on 2026-06-26. All 4 IDs confirmed present in upstream `threejs.org/examples/files.json` on 2026-06-26.

**Already-registered postprocessing demos verified non-overlap** (registry.ts lines 539-592, 47 entries): includes `_3dlut`, `_advanced`, `_masking`, `_fxaa`, `_ssr`, `_pixel`, `_taa`, `_nodes`, `_pixel_art`, `_sao`, `_sobel`, `_unreal_bloom`, `_screenspace` (Phase 16), and many more. None of this plan's IDs collide.

**Note on dropped candidates**: `webgl_postprocessing_3d_lut` from the seed list maps to upstream `webgl_postprocessing_3dlut` which is already registered (line 580). `webgl_postprocessing_stencil` is not in upstream `files.json`. Both excluded. Replaced with `_procedural` and `_unreal_bloom_selective` which are both confirmed unregistered and upstream-present.

## Tasks

### Task 1: Port WebGLPostprocessingBackgrounds

**File**: `demo/src/WebGLPostprocessingBackgrounds.tsx`
**Source**: threejs.org/examples/#webgl_postprocessing_backgrounds
**Key features**:
- `EffectComposer` with `RenderPass` first
- Custom `ShaderPass` that composites a separately-rendered background (equirect or cube env) under the rendered scene
- The "background pass" demonstrates a way to render the env independently of `scene.background` so postprocessing can be applied selectively (e.g., bloom only on the foreground)
- Scene: a foreground object (TorusKnot/sphere) plus an HDR env loaded via `RGBELoader` + `PMREMGenerator`
- `OutputPass` last
- Animation: rotate the foreground object; OrbitControls active
- Resize updates composer + uniforms
**Registry ID**: `webgl_postprocessing_backgrounds`
**Category**: `postprocessing`

### Task 2: Port WebGLPostprocessingRGBHalftone

**File**: `demo/src/WebGLPostprocessingRGBHalftone.tsx`
**Source**: threejs.org/examples/#webgl_postprocessing_rgb_halftone
**Key features**:
- `EffectComposer` + `RenderPass` for the base scene (a textured/MeshStandardMaterial object — rotating sphere or torus)
- `HalftonePass` from `three/examples/jsm/postprocessing/HalftonePass.js` configured for RGB channel-separated dot patterns: `params: { shape: 1, radius: 4, rotateR: ..., rotateG: ..., rotateB: ..., scatter: 0, blending: 1, blendingMode: 1, greyscale: false, disable: false }`
- `OutputPass` last
- GUI (lil-gui) to tweak `radius`, `scatter`, `shape`, per-channel rotation
- OrbitControls on base camera
- Resize updates composer + halftone uniforms
**Registry ID**: `webgl_postprocessing_rgb_halftone`
**Category**: `postprocessing`

### Task 3: Port WebGLPostprocessingProcedural

**File**: `demo/src/WebGLPostprocessingProcedural.tsx`
**Source**: threejs.org/examples/#webgl_postprocessing_procedural
**Key features**:
- This is the "fullscreen procedural shader" pattern — no scene render needed; `EffectComposer` has just a single `ShaderPass` that runs a procedural fragment shader (e.g., animated noise/fractal/gradient pattern) writing directly into the output
- `ShaderMaterial`-style shader object: `uniforms: { time, resolution }`, vertex shader is the standard fullscreen-quad UV passthrough, fragment shader is whatever procedural pattern the upstream example uses (verify against the upstream source)
- Animation loop updates `time` uniform; resize updates `resolution` uniform and `composer.setSize`
- No OrbitControls (no scene to control)
- Minimal `Scene` and `Camera` still required by EffectComposer constructor but they're effectively unused
**Registry ID**: `webgl_postprocessing_procedural`
**Category**: `postprocessing`

### Task 4: Port WebGLPostprocessingUnrealBloomSelective

**File**: `demo/src/WebGLPostprocessingUnrealBloomSelective.tsx`
**Source**: threejs.org/examples/#webgl_postprocessing_unreal_bloom_selective
**Key features**:
- Two `EffectComposer` instances OR layered passes with a mask: the canonical upstream approach uses `Layers` — bloom layer (1) plus the default (0)
- `UnrealBloomPass` from `three/examples/jsm/postprocessing/UnrealBloomPass.js`
- First pass: render only `bloomLayer` objects into a bloom-only composer (with `RenderPass` + `UnrealBloomPass`)
- Second pass: render the full scene with a `ShaderPass` that additively blends in the bloom-composer's output texture as a uniform
- Scene: several spheres at random positions; clicking (or pre-toggled subset) enables the bloom layer on certain spheres so only those glow
- `OrbitControls`
- GUI: bloom threshold/strength/radius
- Resize updates both composers
**Registry ID**: `webgl_postprocessing_unreal_bloom_selective`
**Category**: `postprocessing`
