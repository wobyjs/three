# Phase 17 Plan 02: GLTF Advanced Loaders Batch

## Context

Middle plan of Phase 17 (3 plans total). 17-01 covers 4 GLTF material extensions (iridescence/sheen/transmission/dispersion). This plan groups 4 GLTF loader demos that exercise non-material extensions: animation pointer, progressive LOD, AVIF textures, and EXT_mesh_gpu_instancing. They share the GLTFLoader infrastructure but each requires different setup (animation mixer, LOD switching, AVIF decoder support, instanced-mesh handling) â€” grouping them keeps GLTFLoader knowledge concentrated in one execution session.

All 4 IDs verified absent from `demo/src/registry.ts` on 2026-06-26. All 4 IDs confirmed present in upstream `threejs.org/examples/files.json` on 2026-06-26.

**Already-registered GLTF demos verified non-overlap** (registry.ts lines 500-503, 528-531): `webgl_loader_gltf`, `_anisotropy`, `_demo`, `_2`, `_variants`, `_compressed`, `_lights`, `_browser`. None collide with this plan.

## Tasks

### Task 1: Port WebGLLoaderGLTFAnimationPointer

**File**: `demo/src/WebGLLoaderGLTFAnimationPointer.tsx`
**Source**: threejs.org/examples/#webgl_loader_gltf_animation_pointer
**Key features**:
- `GLTFLoader` loading the upstream `KHR_animation_pointer` sample model (verify current URL from upstream source â€” e.g. `https://threejs.org/examples/models/gltf/AnimationPointerUVs.glb` or similar)
- `AnimationMixer` driven from the loaded `gltf.animations[]` â€” KHR_animation_pointer permits animating arbitrary material/light/camera properties via JSON pointer paths
- Per-frame `mixer.update(delta)` driven by woby effect using `Clock`
- `RGBELoader` + `PMREMGenerator` for environment; `ACESFilmicToneMapping`
- `OrbitControls`
- Cleanup: `mixer.stopAllAction()`, dispose loaders/PMREM
**Registry ID**: `webgl_loader_gltf_animation_pointer`
**Category**: `loaders`

### Task 2: Port WebGLLoaderGLTFProgressiveLOD

**File**: `demo/src/WebGLLoaderGLTFProgressiveLOD.tsx`
**Source**: threejs.org/examples/#webgl_loader_gltf_progressive_lod
**Key features**:
- `GLTFLoader` with `MSFT_lod` extension support (or upstream's progressive-LOD streaming pattern â€” check the upstream demo source for which mechanism)
- Two-stage load: low-detail proxy displayed immediately, high-detail mesh swapped in once secondary fetch resolves
- `THREE.LOD` instance switching between proxies, OR direct mesh swap once high-detail load completes â€” match upstream behavior
- HDR env + PMREMGenerator; OrbitControls
- GUI or on-screen text showing load progress (low vs high asset state)
- Cleanup: dispose all mesh geometries/materials, both proxy and high-detail
**Registry ID**: `webgl_loader_gltf_progressive_lod`
**Category**: `loaders`

### Task 3: Port WebGLLoaderGLTFAvif

**File**: `demo/src/WebGLLoaderGLTFAvif.tsx`
**Source**: threejs.org/examples/#webgl_loader_gltf_avif
**Key features**:
- `GLTFLoader` loading a model whose embedded textures use `KHR_texture_avif` (verify upstream URL â€” e.g. `https://threejs.org/examples/models/gltf/AvifTest.glb` or similar)
- AVIF decode happens natively in browsers that support it (Chrome 85+, Firefox 93+) â€” no explicit decoder setup needed for GLTFLoader if browser supports it; if not, fall back gracefully with a console.warn
- HDR env + PMREMGenerator; ACESFilmicToneMapping
- OrbitControls
- Cleanup: dispose loaders/PMREM, scene contents
**Registry ID**: `webgl_loader_gltf_avif`
**Category**: `loaders`

### Task 4: Port WebGLLoaderGLTFInstancing

**File**: `demo/src/WebGLLoaderGLTFInstancing.tsx`
**Source**: threejs.org/examples/#webgl_loader_gltf_instancing
**Key features**:
- `GLTFLoader` with `GLTFMeshGpuInstancingExtension` (from `three/examples/jsm/loaders/gltf/GLTFMeshGpuInstancingExtension.js`) registered via `loader.register(parser => new GLTFMeshGpuInstancingExtension(parser))`
- Loads a model using `EXT_mesh_gpu_instancing` â€” the loader emits `InstancedMesh` objects in place of regular Meshes
- HDR env via RGBELoader + PMREMGenerator; ACESFilmicToneMapping
- OrbitControls (framed wide enough to show the instanced array)
- Optional rotation per-frame for the InstancedMesh(es)
- Cleanup: dispose InstancedMesh geometries/materials, loaders, PMREM
**Registry ID**: `webgl_loader_gltf_instancing`
**Category**: `loaders`
