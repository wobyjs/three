# Phase 17 Plan 01: GLTF Material Extensions Batch

## Context

First plan of Phase 17. Phase 16 ended at ~691 registered demos. This plan ports 4 GLTF material-extension demos. All 4 IDs verified absent from `demo/src/registry.ts` on 2026-06-26. All 4 IDs confirmed present in upstream `threejs.org/examples/files.json` on 2026-06-26.

These four demos share a common loading pipeline (GLTFLoader + RGBELoader environment + PMREMGenerator + OrbitControls) and only differ in which `KHR_materials_*` extension drives the visual â€” keeping them in one plan keeps the executor in the same mental model for the whole pass.

**Already-registered GLTF demos verified non-overlap** (registry.ts lines 500-503, 528-531): `webgl_loader_gltf`, `_anisotropy`, `_demo`, `_2`, `_variants`, `_compressed`, `_lights`, `_browser`. None of this plan's IDs collide.

**Note on `webgl_loader_gltf_clearcoat`**: NOT in upstream `files.json` (verified 2026-06-26) â€” clearcoat is part of the core glTF 2.0 PBR spec since 2019 and no longer has a dedicated demo upstream. Therefore not included here.

## Tasks

### Task 1: Port WebGLLoaderGLTFIridescence

**File**: `demo/src/WebGLLoaderGLTFIridescence.tsx`
**Source**: threejs.org/examples/#webgl_loader_gltf_iridescence
**Key features**:
- `GLTFLoader` from `three/examples/jsm/loaders/GLTFLoader.js` loading the `IridescenceLamp.glb` model (or current upstream URL such as `https://threejs.org/examples/models/gltf/IridescenceLamp.glb`)
- `RGBELoader` loading an HDR environment (`venice_sunset_1k.hdr` or current upstream URL)
- `PMREMGenerator` to pre-filter the HDR into a usable PMREM env; assign to `scene.environment` and (optionally) `scene.background`
- `ACESFilmicToneMapping` on the renderer, `renderer.toneMappingExposure = 1.0`
- `OrbitControls` (auto-rotate off by default, target framed on the model)
- The GLTF embeds `KHR_materials_iridescence` so visual goal is the thin-film color shift as the camera orbits â€” no extra material tweaking needed
- Cleanup: dispose loaders, PMREMGenerator, geometries/materials in scene
**Registry ID**: `webgl_loader_gltf_iridescence`
**Category**: `loaders`

### Task 2: Port WebGLLoaderGLTFSheen

**File**: `demo/src/WebGLLoaderGLTFSheen.tsx`
**Source**: threejs.org/examples/#webgl_loader_gltf_sheen
**Key features**:
- `GLTFLoader` loading `SheenChair.glb` (or current upstream URL such as `https://threejs.org/examples/models/gltf/SheenChair.glb`)
- `RGBELoader` + `PMREMGenerator` environment (any neutral HDR â€” `venice_sunset_1k.hdr` is fine)
- `ACESFilmicToneMapping`, exposure ~1.0
- `OrbitControls` framed on the chair
- The GLTF embeds `KHR_materials_sheen` â€” visual goal is the soft velvet rim-light as the camera orbits
- Optional minimal GUI to toggle sheen on/off by mutating `material.sheen` on the loaded mesh's materials
- Cleanup: same as Task 1
**Registry ID**: `webgl_loader_gltf_sheen`
**Category**: `loaders`

### Task 3: Port WebGLLoaderGLTFTransmission

**File**: `demo/src/WebGLLoaderGLTFTransmission.tsx`
**Source**: threejs.org/examples/#webgl_loader_gltf_transmission
**Key features**:
- `GLTFLoader` loading `IridescentDishWithOlives.glb` or `TransmissionTest.glb` (use whichever upstream demo currently references â€” check the upstream source file)
- HDR env via `RGBELoader` + `PMREMGenerator`, also set as `scene.background` so refraction is visible
- `renderer.toneMapping = ACESFilmicToneMapping`, `renderer.toneMappingExposure = 1.0`
- Critically: transmission rendering needs `renderer.localClippingEnabled` not required, but the loaded materials should have `transmission > 0`, `thickness > 0`, `ior` set â€” handled by the GLTF, no manual material edits needed
- `OrbitControls`
- Cleanup: dispose loaders, PMREM, scene contents
**Registry ID**: `webgl_loader_gltf_transmission`
**Category**: `loaders`

### Task 4: Port WebGLLoaderGLTFDispersion

**File**: `demo/src/WebGLLoaderGLTFDispersion.tsx`
**Source**: threejs.org/examples/#webgl_loader_gltf_dispersion
**Key features**:
- `GLTFLoader` loading `DispersionTest.glb` (or current upstream URL â€” verify against the upstream demo source)
- `RGBELoader` HDR env + `PMREMGenerator`, also as `scene.background`
- `ACESFilmicToneMapping`, exposure ~1.0
- `OrbitControls`
- The GLTF embeds `KHR_materials_dispersion` together with `KHR_materials_transmission` â€” visual goal is the rainbow chromatic fringing at glass edges as the camera orbits

**Registry ID**: `webgl_loader_gltf_dispersion`
