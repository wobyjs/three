# Phase 18 Plan 02: Texture Sampling + Transform Batch

## Context

Middle plan of Phase 18 (3 plans total). 18-01 covers HDR/EXR env-map reflection and shader injection. This plan groups 4 texture-state demos that all exercise different facets of `THREE.Texture` configuration: anisotropic max-quality filtering, the mag/min filter dropdown, UV-space rotation via `texture.center`/`texture.rotation`, and using a live HTML DOM element as a texture source via `CanvasTexture`/`html2canvas`. Grouped together because they share the same `TextureLoader` + `MeshBasicMaterial({ map: texture })` foundation and only differ in which texture-property is the focus.

All 4 IDs verified absent from `demo/src/registry.ts` on 2026-06-26 (grep returned 0 matches for each). All 4 IDs confirmed present in upstream `threejs.org/examples/files.json` on 2026-06-26.

**Already-registered texture demos verified non-overlap** (registry.ts lines 276, 301, 302): `webgl_materials_texture_grid` (line 276 — uses `WebGLMaterialsTextureGuid`), `webgl_materials_texture` (line 301), `webgl_materials_texture_canvas` (line 302 — uses `WebGLTextureCanvas2`). None of this plan's 4 IDs collide.

**Pre-converted stub note**: The four `.tsx` files under `code/examples/webgl/materials/` (`MaterialsTextureAnisotropy.tsx`, `MaterialsTextureFilters.tsx`, `MaterialsTextureRotation.tsx`, `MaterialsTextureHtml.tsx`) are scaffolding-only. None of them actually configure `texture.anisotropy`, set distinct min/mag filters, animate `texture.rotation`, or rasterize HTML. The implementer MUST consult upstream `threejs.org/examples/#<id>` for canonical behavior.

## Tasks

### Task 1: Port WebGLMaterialsTextureAnisotropy

**File**: `demo/src/WebGLMaterialsTextureAnisotropy.tsx`
**Source**: threejs.org/examples/#webgl_materials_texture_anisotropy
**Reference scaffold**: `code/examples/webgl/materials/MaterialsTextureAnisotropy.tsx` (stub only — does not configure anisotropy)
**Key features**:
- A long ground/floor plane viewed at a glancing angle so the anisotropic-filtering effect is visible — upstream uses a `PlaneGeometry(100, 100)` rotated `-PI/2` on X with a repeating `crate.gif` or `disturb.jpg` texture (`texture.wrapS = texture.wrapT = THREE.RepeatWrapping`, `texture.repeat.set(512, 512)`)
- Two side-by-side viewports OR a split-screen via two `WebGLRenderer.setScissor` regions: left half uses `texture.anisotropy = 1` (baseline trilinear), right half uses `texture.anisotropy = renderer.capabilities.getMaxAnisotropy()` (typically 16). Visual goal: the high-anisotropy half shows crisp texels at the receding horizon, the low-anisotropy half blurs to grey
- Alternative simpler upstream pattern: single viewport with two planes side-by-side, each using a separately-cloned texture with different `anisotropy` values
- `PerspectiveCamera` positioned low and angled down to maximize the glancing-angle effect
- `OrbitControls` to let the user pan around
- HUD/GUI showing current anisotropy values for each half
- Cleanup: dispose textures, geometries, materials, controls, GUI, canvas
**Registry ID**: `webgl_materials_texture_anisotropy`
**Registry name**: `'Texture Anisotropy (Glancing-Angle Filtering)'`
**Category**: `materials`

### Task 2: Port WebGLMaterialsTextureFilters

**File**: `demo/src/WebGLMaterialsTextureFilters.tsx`
**Source**: threejs.org/examples/#webgl_materials_texture_filters
**Reference scaffold**: `code/examples/webgl/materials/MaterialsTextureFilters.tsx` (stub creates 2 planes with NearestFilter vs LinearFilter — good direction, but upstream covers more filter modes; expand to match)
**Key features**:
- Multiple planes (typically 4–6) each displaying the same checkerboard or pixelated texture but with different `texture.minFilter` / `texture.magFilter` combinations: `NearestFilter`, `LinearFilter`, `NearestMipmapNearestFilter`, `LinearMipmapLinearFilter`, etc.
- Procedural checkerboard generated via `CanvasTexture` (canvas 64×64 with 8×8 black/white grid) — same pattern the stub already uses, but expand the number of filter variants
- Each plane labeled (text sprite or HTML overlay) with its filter mode name so the user can identify which is which
- `OrbitControls` to let user zoom/orbit and see how each filter reacts to minification (zoom out → see mipmap quality) vs magnification (zoom in → see pixelation vs blur)
- Cleanup: dispose all textures, geometries, materials, controls, canvas
**Registry ID**: `webgl_materials_texture_filters`
**Registry name**: `'Texture Filter Modes'`
**Category**: `materials`

### Task 3: Port WebGLMaterialsTextureRotation

**File**: `demo/src/WebGLMaterialsTextureRotation.tsx`
**Source**: threejs.org/examples/#webgl_materials_texture_rotation
**Reference scaffold**: `code/examples/webgl/materials/MaterialsTextureRotation.tsx` (stub rotates the MESH, not the texture — incorrect direction; rewrite to animate `texture.rotation`)
**Key features**:
- A static (non-rotating) plane displaying a textured image (`crate.gif` or `uv_grid.jpg`)
- The TEXTURE rotates, not the mesh: animate `texture.rotation = clock.getElapsedTime() * 0.5` per-frame
- `texture.center.set(0.5, 0.5)` to rotate around the texture's center rather than the corner
- Optional: GUI controls for `texture.center.x`, `texture.center.y`, `texture.offset.x`, `texture.offset.y`, `texture.repeat.x`, `texture.repeat.y` so the user can experiment with the full UV transform
- `MeshBasicMaterial({ map: texture })`
- `OrbitControls` (orbit the camera, the mesh stays put, the texture spins inside the UVs)
- Cleanup: dispose texture, geometry, material, controls, GUI, canvas
**Registry ID**: `webgl_materials_texture_rotation`
**Registry name**: `'Texture UV Rotation'`
**Category**: `materials`

### Task 4: Port WebGLMaterialsTextureHTML

**File**: `demo/src/WebGLMaterialsTextureHTML.tsx`
**Source**: threejs.org/examples/#webgl_materials_texture_html
**Reference scaffold**: `code/examples/webgl/materials/MaterialsTextureHtml.tsx` (stub displays an untextured colored plane — completely misses the HTML-to-texture concept)
**Key features**:
- Render a live HTML DOM element (typically a styled `<div>` containing text + colored backgrounds + images) into a `CanvasTexture` via `html2canvas` or `foreignObject + SVG → image` blob URL approach
- If `html2canvas` is the upstream choice: `npm i html2canvas --workspace demo` (verify the package on npmjs.com/package/html2canvas first — legitimate, 2M+ weekly DLs, well-maintained)
- Upstream typically constructs an off-screen `<div>` styled with `position: absolute; left: -9999px;` containing the content, calls `html2canvas(div)` → returns a canvas → wrap with `new CanvasTexture(canvas)` → assign to `MeshBasicMaterial.map`
- Apply the texture to a `PlaneGeometry` or `BoxGeometry` cube so the HTML content is visible as a 3D textured surface
- Animation: rotate the mesh slowly so the textured surface is visible from multiple angles
- Optional: re-rasterize the HTML every N seconds (or on user click) to show live updates of dynamic DOM content
- `OrbitControls`
- Cleanup: remove the off-screen `<div>` from DOM, dispose texture/geometry/material/controls/canvas
**Registry ID**: `webgl_materials_texture_html`
**Registry name**: `'HTML Element → Texture'`
**Category**: `materials`
