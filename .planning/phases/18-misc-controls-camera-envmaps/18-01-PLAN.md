### Task 4: Port WebGLMaterialsModified

**File**: `demo/src/WebGLMaterialsModified.tsx`
**Source**: threejs.org/examples/#webgl_materials_modified
**Reference scaffold**: `code/examples/webgl/materials/MaterialsModified.tsx` (stub renders 3 spheres with off-the-shelf materials â€” does NOT demonstrate `onBeforeCompile`)
**Key features**:
- This is the canonical `Material.onBeforeCompile` shader-injection demo â€” start from a stock `MeshStandardMaterial` and inject custom GLSL into the vertex shader to displace vertices procedurally
- Upstream pattern:
  ```
  material.onBeforeCompile = (shader) => {
    shader.uniforms.time = { value: 0 }
    shader.vertexShader = 'uniform float time;\n' + shader.vertexShader
    shader.vertexShader = shader.vertexShader.replace(
      '#include <begin_vertex>',
      `#include <begin_vertex>
       transformed.x += sin(transformed.y * 5.0 + time) * 0.1;`
    )
    material.userData.shader = shader
  }
  ```
- Per-frame: `if (material.userData.shader) material.userData.shader.uniforms.time.value = clock.getElapsedTime()`
- Geometry: upstream uses a loaded GLTF model (often `LeePerrySmith.glb` or a `BufferGeometry`-heavy mesh) so the displacement is visible across many vertices. If GLTF load is heavy, an `IcosahedronGeometry(1, 8)` (high subdivision) is an acceptable substitute that still showcases per-vertex shader displacement â€” document any substitution in SUMMARY.
- `HemisphereLight` + `DirectionalLight` (or env from PMREM â€” match upstream)
- `OrbitControls`
- Animation loop drives the `time` uniform
- Cleanup: dispose geometry/material/controls/canvas; the patched shader cleans up when the material does
**Registry ID**: `webgl_materials_modified`
**Registry name**: `'Modified Material (onBeforeCompile)'`
**Category**: `materials`
