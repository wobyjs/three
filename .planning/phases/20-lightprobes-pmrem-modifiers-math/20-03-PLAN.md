### Task 1: Port WebGLModifierCurveInstanced

**File**: `demo/src/WebGLModifierCurveInstanced.tsx`
**Source**: https://threejs.org/examples/#webgl_modifier_curve_instanced
**Reference scaffold**: `code/examples/webgl/modifiers/ModifierCurveInstanced.tsx` (stub — does not use CurveModifier or InstancedMesh)

**Step-by-step workflow:**
1. `WebFetch` upstream HTML: `https://threejs.org/examples/webgl_modifier_curve_instanced.html` — extract canonical: CurveModifier import path, curve construction, instance count, geometry flow-along-curve setup.
2. Read scaffold: `code/examples/webgl/modifiers/ModifierCurveInstanced.tsx`.
3. Write `demo/src/WebGLModifierCurveInstanced.tsx` (init3D pattern). Required elements:
   - `/** @jsxImportSource woby */`
   - Imports: `THREE`, `{ Flow, InstancedFlow }` from `three/examples/jsm/modifiers/CurveModifier.js`, `OrbitControls`, `FontLoader` if upstream uses text geometry (otherwise omit)
   - Construct a curve: typically `new THREE.CatmullRomCurve3([...vec3 points...])` with ~6 control points forming a 3D loop
   - Visualize the curve: `new THREE.Line(new THREE.BufferGeometry().setFromPoints(curve.getPoints(100)), new THREE.LineBasicMaterial({ color: 0x00ffff }))`
   - Base geometry for instancing: small box or arrow (`BoxGeometry(0.1, 0.1, 0.3)`)
   - Create `InstancedFlow(count = ~20, curveCount = 1, geometry, material)` — `InstancedFlow` is the instanced variant of `Flow` from CurveModifier.js
   - `flow.updateCurve(0, curve)` to bind the curve
   - For each instance: `flow.setCurve(instanceIndex, 0); flow.moveIndividualAlongCurve(instanceIndex, initialOffset)` so they spread out along the path
   - Per-frame: `for each instance: flow.moveIndividualAlongCurve(i, 0.001)` to animate them flowing along
   - `flow.object3D.castShadow = true` (or whatever upstream uses)
   - Scene: add `flow.object3D` and the curve visualization line
   - `OrbitControls`, `HemisphereLight` + `DirectionalLight` for visibility
   - Cleanup: dispose base geometry+material, curve line geometry+material, controls, canvas. CurveModifier `Flow` instance has no `dispose()` — remove `flow.object3D` from scene
4. Typecheck: `pnpm --filter demo exec tsc --noEmit --skipLibCheck`
5. dv visual verify on profile-4: navigate `#webgl_modifier_curve_instanced` → wait 2s (no async assets) → screenshot → Claude confirms: curve line visible AND multiple instanced boxes positioned along the curve at distinct points (NOT all clustered at origin — that means `moveIndividualAlongCurve` wasn't called)
6. ONLY AFTER visual approval: append registry entry

### Task 2: Port WebGLModifierSimplifier

**File**: `demo/src/WebGLModifierSimplifier.tsx`
**Source**: https://threejs.org/examples/#webgl_modifier_simplifier
**Reference scaffold**: `code/examples/webgl/modifiers/ModifierSimplifier.tsx` (stub — does not apply SimplifyModifier)

**Step-by-step workflow:**
1. `WebFetch` upstream HTML: `https://threejs.org/examples/webgl_modifier_simplifier.html` — extract canonical: source model URL (often `LeePerrySmith.glb` or similar), simplify ratio, side-by-side comparison setup.
2. Read scaffold: `code/examples/webgl/modifiers/ModifierSimplifier.tsx`.
3. Write `demo/src/WebGLModifierSimplifier.tsx` (init3D pattern). Required elements:
   - `/** @jsxImportSource woby */`
   - Imports: `THREE`, `{ SimplifyModifier }` from `three/examples/jsm/modifiers/SimplifyModifier.js`, `GLTFLoader` (if loading external model), `OrbitControls`
   - Load a source mesh: upstream typically uses `LeePerrySmith.glb` from `https://threejs.org/examples/models/gltf/LeePerrySmith/LeePerrySmith.glb`. If load fails, fall back to `IcosahedronGeometry(1, 4)` (high-tessellation procedural sphere) — note in SUMMARY
   - On geometry-ready: `const modifier = new SimplifyModifier(); const simplified = modifier.modify(geometry, Math.floor(geometry.attributes.position.count * 0.25))` — keep 25% of vertices
   - Display BOTH meshes side-by-side: left mesh = original geometry, right mesh = simplified. Use `MeshNormalMaterial` so the topology difference is highly visible (low-poly faceting on the right, smooth on the left)
   - Add lighting: `DirectionalLight` + ambient
   - lil-gui: `ratio` slider (0.05 – 1.0) that on-change re-runs `modifier.modify(originalGeometry, Math.floor(count * ratio))` and replaces the right mesh's geometry. The previous simplified geometry MUST be `.dispose()`ed before assignment to avoid GPU memory leak
   - `OrbitControls`
   - Cleanup: dispose both geometries (original + current simplified), material, controls, GUI, canvas
4. Typecheck: `pnpm --filter demo exec tsc --noEmit --skipLibCheck`
5. dv visual verify on profile-4: navigate `#webgl_modifier_simplifier` → wait 4s (GLTF load) → screenshot → Claude confirms: TWO meshes visible side-by-side, right one VISIBLY lower-poly (faceting/triangle bands visible)

### Task 4: Port WebGLMathOBB

**File**: `demo/src/WebGLMathOBB.tsx` (preserve OBB uppercase per Three.js `OBB` class convention)
**Source**: https://threejs.org/examples/#webgl_math_obb
**Reference scaffold**: `code/examples/webgl/math/MathObb.tsx` (stub — does not instantiate OBB or perform collision tests)

**Step-by-step workflow:**
1. `WebFetch` upstream HTML: `https://threejs.org/examples/webgl_math_obb.html` — extract canonical: how OBB is constructed (from geometry or from explicit center+halfSize+rotation), collision-test loop, color-on-collision pattern.
2. Read scaffold: `code/examples/webgl/math/MathObb.tsx`.
3. Write `demo/src/WebGLMathOBB.tsx` (init3D pattern). Required elements:
   - `/** @jsxImportSource woby */`
   - Imports: `THREE`, `{ OBB }` from `three/examples/jsm/math/OBB.js`, `OrbitControls`
   - Create N boxes (e.g. 50): random `BoxGeometry` with random size, random position, random rotation. Each box gets a `userData.obb = new OBB(center, halfSize)` and `geometry.userData.obb = obb.clone()` stored once at creation
   - Per-frame loop: rotate one "selected" box (or rotate all slowly). For each box, update its OBB: `box.userData.obb.copy(geometry.userData.obb).applyMatrix4(box.matrixWorld)`
   - Collision test: for each pair (i, j) where i < j: `if (boxes[i].userData.obb.intersectsOBB(boxes[j].userData.obb)) { /* mark both colliding */ }`
   - Visualize collision: set material color red on colliding boxes, green/blue on non-colliding (use shared materials or per-box materials, clone if needed)
   - `OrbitControls`, animation loop (boxes rotate slowly so collisions transition in/out and the user sees colors change)
   - Add a single big box that the user can drag (or that rotates) to demonstrate dynamic OBB updates clearly
   - Cleanup: dispose all geometries + materials (one per box if per-box, or shared pair), controls, canvas
4. Typecheck: `pnpm --filter demo exec tsc --noEmit --skipLibCheck`
5. dv visual verify on profile-4: navigate `#webgl_math_obb` → wait 3s (animation needs a frame or two so colors update from initial state) → screenshot → Claude confirms: multiple oriented boxes visible with VISIBLY DIFFERENT colors (red = colliding, green = not), not all uniform
6. ONLY AFTER visual approval: append registry entry
