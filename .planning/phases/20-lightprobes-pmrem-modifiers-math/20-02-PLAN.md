### Task 3: Port WebGLMaterialsCubemapMipmaps

**File**: `demo/src/WebGLMaterialsCubemapMipmaps.tsx`
**Source**: https://threejs.org/examples/#webgl_materials_cubemap_mipmaps
**Reference scaffold**: `code/examples/webgl/materials/MaterialsCubemapMipmaps.tsx` (stub â€” does not author per-mip canvases)

**Step-by-step workflow:**
1. `WebFetch` upstream HTML: `https://threejs.org/examples/webgl_materials_cubemap_mipmaps.html` â€” extract canonical: cube texture base, mipmap loading strategy, distinguishing colors per mip.
2. Read scaffold: `code/examples/webgl/materials/MaterialsCubemapMipmaps.tsx`.
3. Write `demo/src/WebGLMaterialsCubemapMipmaps.tsx` (init3D pattern). Required elements:
   - `/** @jsxImportSource woby */`
   - Imports: `THREE`, `OrbitControls`, `CubeTextureLoader`
   - Strategy: load base cube texture (e.g. `pisa/`), then manually populate `cubeTexture.mipmaps = [level0Cubes, level1Cubes, level2Cubes, ...]` where each `levelN` is an array of 6 canvas faces. Upstream typically TINTS each successive mip level with a distinct color overlay (e.g. mip0 = natural, mip1 = red tint, mip2 = green tint, mip3 = blue tint, ...) so when the renderer samples a smaller mip the user visibly sees the tint change
   - For each mip level N (N = 0 â€¦ 5, sizes 256 â†’ 128 â†’ 64 â†’ 32 â†’ 16 â†’ 8 â†’ 4 â†’ 2 â†’ 1):
     - Create 6 OffscreenCanvas / HTMLCanvasElement of size `256 >> N` each
     - Draw downsampled base + a tint overlay color per level
   - `cubeTexture.generateMipmaps = false` (critical â€” otherwise WebGL auto-generates and overwrites)
   - `cubeTexture.minFilter = THREE.LinearMipmapLinearFilter; cubeTexture.magFilter = THREE.LinearFilter`
   - `scene.background = cubeTexture` so the user sees the cubemap directly (and watches the tint shift as they zoom/move)
   - Add a reflective `MeshBasicMaterial({ envMap: cubeTexture })` sphere or skybox subject so the mip selection is visible from BOTH background and reflection sampling
   - `OrbitControls` (zoom in â†’ see mip0 sharp natural; zoom out â†’ see mip3+ blue/green tinted)
   - Cleanup: dispose cube texture (and all level canvases via GC), geometry, material, controls, canvas
4. Typecheck: `pnpm --filter demo exec tsc --noEmit --skipLibCheck`
5. dv visual verify on profile-4: navigate `#webgl_materials_cubemap_mipmaps` â†’ wait 4s â†’ screenshot â†’ Claude confirms cube texture visible AND distinct color bands visible (the per-mip tints) â€” if all levels look identical, `generateMipmaps = false` was forgotten and the chain was overwritten
6. ONLY AFTER visual approval: append registry entry

**File**: `demo/src/WebGLMaterialsCubemapRenderToMipmaps.tsx`
**Source**: https://threejs.org/examples/#webgl_materials_cubemap_render_to_mipmaps
**Reference scaffold**: `code/examples/webgl/materials/MaterialsCubemapRenderToMipmaps.tsx` (stub â€” does not perform per-mip render)

   - Create a `WebGLCubeRenderTarget(256, { generateMipmaps: false, minFilter: THREE.LinearMipmapLinearFilter, magFilter: THREE.LinearFilter })`
