# Phase 20 Plan 01: Light Probes + Cubemap PMREM Batch

## Context

First plan of Phase 20 (3 plans total). Phase 18 closed at verifier PASS 12/12, Phase 19 is planned in parallel. This plan groups the 3 LightProbe demos (which all share `LightProbeGenerator` from `three/examples/jsm/lights/LightProbeGenerator.js` + spherical-harmonics coefficient capture from a `CubeTexture` / `WebGLCubeRenderTarget`) with the first PMREM demo (`pmrem_cubemap`) which uses `PMREMGenerator.fromCubemap()` to pre-filter a cube texture into a roughness-mipped PMREM cube. All four exercise the cube-texture → IBL pipeline and use `MeshStandardMaterial` + `ACESFilmicToneMapping`.

The 3 LightProbe demos progress in complexity: bare LightProbe over a flat sphere → structured scene with multiple meshes → full Sponza GLTF with both diffuse and specular probes.

### Task 1: Port WebGLLightProbes

**File**: `demo/src/WebGLLightProbes.tsx`
**Source**: https://threejs.org/examples/#webgl_lightprobes
**Reference scaffold**: `code/examples/webgl/lights/LightProbes.tsx` (stub — does not configure LightProbeGenerator or load a cube texture; rewrite to canonical)

**Step-by-step workflow:**
1. `WebFetch` upstream HTML: `https://threejs.org/examples/webgl_lightprobes.html` — extract canonical: cube texture path, LightProbeGenerator usage, scene composition.
2. Read scaffold: `code/examples/webgl/lights/LightProbes.tsx` for naming conventions and import paths already wired.
3. Write `demo/src/WebGLLightProbes.tsx` (full rewrite, init3D container-ref pattern). Required elements:
   - `/** @jsxImportSource woby */` first line
   - Imports: `THREE` from `three`, `LightProbeGenerator` from `three/examples/jsm/lights/LightProbeGenerator.js`, `OrbitControls` from `three/examples/jsm/controls/OrbitControls.js`, `CubeTextureLoader` (built-in)

   - `scene.background = cubeTexture` so the user sees the source environment
   - `renderer.toneMapping = THREE.NoToneMapping` (upstream default for the bare probe demo; if upstream uses ACESFilmic, match it)
   - `OrbitControls` framed on the sphere
   - Animation loop renders; lil-gui slider for `lightProbe.intensity` (0–2) so the probe contribution is interactively visible
   - Cleanup: dispose cubeTexture, sphere geometry+material, controls, GUI, remove canvas
4. Typecheck: `pnpm --filter demo exec tsc --noEmit --skipLibCheck` from repo root (must pass clean)
5. dv visual verify on profile-4 (recipe in "dv CLI verification recipe" section below):

**Source**: https://threejs.org/examples/#webgl_pmrem_cubemap
**Reference scaffold**: `code/examples/webgl/pmrem/PmremCubemap.tsx` (stub — does not configure PMREMGenerator)

**Step-by-step workflow:**
