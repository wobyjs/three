## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| KIMI_API_KEY -> HTTP request | Key used only in Authorization header of fetch() call — never echoed |
| Kimi response -> report JSON | Kimi verdict JSON written verbatim to report; no code execution |
| report HTML -> browser | HTML generated server-side with escaped content; no XSS vectors |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-14 | Information Disclosure | KIMI_API_KEY in report JSON | mitigate | Report contains only demo IDs, scores, verdicts — key is never written to disk |
| T-14-15 | Tampering | Kimi verdict injection via JSON | mitigate | parseVerdict() uses JSON.parse only; no eval; fields are only written to report |
| T-14-16 | Denial of Service | Kimi rate limiting for 3 workers | accept | If rate-limited, reduce --workers=2 and re-run; partial results are preserved |
| T-14-18 | Tampering | Shell injection via demo ID | accept | Chrome DevTools MCP eliminates shell subprocess — no CLI args parsed by shell |
</threat_model>
