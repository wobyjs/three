Create `playwright.test/scripts/kimi-comparison-worker.ts`. This file is spawned as a child process by the orchestrator. It reads CLI args, processes its assigned batch of demos, and writes partial-{batchIndex}.json.

**CLI args parsed at top (no commander dependency — use process.argv.slice(2)):**
- `--batch-index=N` — required; which batch this worker handles
- `--batch-size=N` — required; how many demos per worker  
- `--dry-run` — if present, skip real Kimi API call; write mock verdicts instead
- `--limit=N` — if present, cap total demos processed to N (for local testing)

**Chrome profile assignment (Phase 14 uses profiles 4-6):**
- Worker 0 → profile-qmdj-4 (port 9225)
- Worker 1 → profile-qmdj-5 (port 9226)
- Worker 2 → profile-qmdj-6 (port 9227)
- Each worker uses ONE profile for both ported and reference captures (sequential within worker)

**Wait time logic:**
- Default wait: 8000 ms
- For demo IDs matching `/^webgl_loader_/`: use 12000 ms (asset-heavy loaders need more time)

**Chrome DevTools MCP capture pattern:**
Workers invoke MCP tools directly (no subprocess):
```typescript
// Profile mapping for workers
const PROFILE_MAP = {
  0: { name: 'profile-qmdj-4', port: 9225 },
  1: { name: 'profile-qmdj-5', port: 9226 },
  2: { name: 'profile-qmdj-6', port: 9227 }
}
const profile = PROFILE_MAP[batchIndex]

// Navigate (headed mode required for WebGL)
await mcp__chrome-devtools__navigate_page({ url, headed: true })

// Wait for canvas element
await mcp__chrome-devtools__wait_for({ text: ["canvas"], timeout: waitMs })

// Take screenshot
await mcp__chrome-devtools__take_screenshot({ filePath: outputPath })

// Check console for errors
const messages = await mcp__chrome-devtools__list_console_messages()
const errors = messages.filter(msg => msg.type === "error" || msg.type === "assert")
```

**Per-demo processing loop:**
For each demo in this worker's slice of ALL_DEMOS (filtered to exclude CUSTOM_DEMO_IDS):
1. Validate demo ID with validateDemoId() — skip with error status if invalid
2. Determine ported screenshot path: `{ROOT}/screenshots/ported/{id}.png`
3. If ported screenshot does not exist on disk, skip re-capture attempt and mark status `no-ported-screenshot`
4. If ported screenshot exists, reuse it (do not re-capture — these exist from Phase 13)
5. Determine reference screenshot path: `{ROOT}/screenshots/reference/{id}.png`
6. If reference screenshot does NOT exist, capture it via Chrome DevTools MCP navigate + wait + screenshot
7. If reference capture fails (console errors or no canvas detected), mark status `reference-load-failed`; continue
8. If reference screenshot exists and ported screenshot exists, call kimiCompare(portedPath, refPath)
9. If dry-run: produce mock Verdict `{ similarity_score: 0.85, verdict: 'pass', reasoning: 'Dry run.', key_differences: [] }`
10. Write result to partial array; append to partial-{batchIndex}.json after each demo (incremental)

const NUM_WORKERS = parseInt(process.argv.find(a => a.startsWith('--workers='))?.split('=')[1] ?? '3', 10)
const batchSize = Math.ceil(comparableDemos.length / NUM_WORKERS)

const workerPromises = Array.from({ length: NUM_WORKERS }, (_, i) => {
  return new Promise<void>((resolve, reject) => {
    const args = [
      'tsx', 'scripts/kimi-comparison-worker.ts',
      `--batch-index=${i}`, `--batch-size=${batchSize}`,
    ]
    if (isDryRun) args.push('--dry-run')
    if (limitArg) args.push(limitArg)

    const child = spawn('npx', args, {
      cwd: ROOT,
      stdio: ['ignore', 'pipe', 'pipe'],
      env: { ...process.env }
    })
    child.stdout.on('data', (d: Buffer) => process.stdout.write(`[worker-${i}] ${d}`))
    child.stderr.on('data', (d: Buffer) => process.stderr.write(`[worker-${i}] ${d}`))
    child.on('close', (code: number | null) =>
      code === 0 ? resolve() : reject(new Error(`Worker ${i} exited with code ${code}`))
    )
  })
})

await Promise.allSettled(workerPromises)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env -> worker processes | KIMI_API_KEY propagated via `env: { ...process.env }` to worker spawns — never logged |
| partial JSON -> merged report | Worker output files are read and JSON.parsed; no eval of file contents |
| demo ID -> session name | Session names are derived from batchIndex (integer), not from demo ID strings |
| demo ID -> file path | validateDemoId() called in worker before any path.join with demo ID |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-06 | Information Disclosure | KIMI_API_KEY in worker env | mitigate | Passed as env object reference only; never logged in orchestrator or worker stdout |
| T-14-07 | Tampering | Chrome profile collision | mitigate | Profiles assigned by worker index (0→profile-4, 1→profile-5, 2→profile-6); no cross-worker overlap |
| T-14-08 | Tampering | Path traversal in partial JSON read | mitigate | Partial file paths constructed from integer batchIndex only, not from external input |
| T-14-09 | Denial of Service | Worker process crash | accept | Promise.allSettled ensures partial results are merged even if some workers fail |
| T-14-10 | Information Disclosure | Dev server pre-flight error message | accept | Error message shows only 'localhost:5175' — no secret data in error output |
