### Task 1: Port WebGLVideoKinect

**File**: `demo/src/WebGLVideoKinect.tsx`
**Source**: `https://threejs.org/examples/#webgl_video_kinect`
**Reference scaffold**: `code/examples/webgl/video/VideoKinect.tsx` (forbidden-pattern stub)
**Key features**:
- Load Microsoft Kinect-format depth+RGB video from `https://threejs.org/examples/textures/kinect.webm` (LOCKED asset URL — verified in CONTEXT)
- The Kinect video encodes RGB on the left half and 8-bit depth on the right half (or upstream may use a different packing — verify against current upstream HTML/JS)
- Wrap the `<video>` with `THREE.VideoTexture(video)` and feed into a custom `ShaderMaterial`:
  - Vertex shader: sample the depth texture (right half UV) per-vertex, displace the vertex along Z by the depth value (scaled appropriately) → builds a point cloud whose geometry matches the depth scene
  - Fragment shader: sample the RGB texture (left half UV) at the corresponding pixel → assign as point color
- Geometry: a dense `BufferGeometry` of `THREE.Points` — one point per source pixel (e.g. 320×240 = 76,800 points for VGA Kinect; or downscale to 160×120 = 19,200 for performance)
- The video plays in a loop (`video.loop = true; video.muted = true; video.play()`); per-frame the VideoTexture auto-updates and the shader re-evaluates → animated 3D point cloud showing the recorded depth scene moving
- `OrbitControls` so user can orbit around the depth scene
- Background: black for contrast against the point cloud
- Cleanup: `video.pause(); video.srcObject = null; document.body.contains(video) && document.body.removeChild(video)`; dispose VideoTexture, shader material, geometry, controls, canvas
- **NO permission prompt** (remote video file load, not getUserMedia) — but autoplay may require `video.muted = true` per Chrome policy
**Registry ID**: `webgl_video_kinect`
**Registry name**: `'Kinect Depth Video → Point Cloud'`
**Category**: `video` (match existing `webgl_video*` rows)
